# Four-Window Dashboard Runtime Data Binding Runbook

## Purpose

Use this runbook to rebuild and publish the integrated Dashboard runtime data binding safely.

## Build

Run:

```bash
./venv/bin/python scripts/orchestrator/build_four_window_dashboard_runtime_data_v1.py --pretty
./venv/bin/python scripts/orchestrator/build_four_window_dashboard_route_preview.py --pretty
```

The builder may open `data/stock_analysis.db` only with read-only SQLite URI mode and must not write, migrate, vacuum, update, delete, insert, or create tables.

## Validate

Run:

```bash
./venv/bin/python scripts/orchestrator/validate_four_window_dashboard_runtime_data_binding_v1.py --pretty
```

After publish, run the same validator with `--published`.

## Publish

Use the existing controlled static preview publish helper:

```bash
./venv/bin/python scripts/orchestrator/publish_four_window_dashboard_preview_v1.py --pretty
```

## Rollback

Use the rollback backup and command generated by the publish manifest. Do not change nginx, systemd, firewall, scheduler, delivery behavior, DB, production pipeline, or credentials.
